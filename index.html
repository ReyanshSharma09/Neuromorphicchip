Neuromorphic Chip Workflow

Complete Pipeline: Input → Spikes → Processing → Output
1

Input Encoding

Convert sensory data into spike trains

Idle

Main Function: Transform continuous analog signals (images, audio, sensor data) into discrete spike events. The chip uses rate coding (higher values = more spikes) or temporal coding (precise timing encodes information).

Input Pattern → Spike Train

2

Neuron State Update

Integrate incoming spikes and update membrane potential

Idle

Main Function: Each neuron accumulates weighted inputs from synapses, maintains membrane voltage with leak dynamics (LIF model), and fires a spike when threshold is reached. Hardware implements this via specialized circuits or efficient digital cores.

Neuron Layer (8 neurons)

3

Event-Driven Spike Routing

Route spike events through synaptic connections

Idle

Main Function: When neurons fire, spike events are routed through a Network-on-Chip (NoC) to target neurons. Computation happens only when spikes occur, dramatically reducing energy compared to continuous processing. Synaptic weights and delays are applied during routing.

On-Chip Learning (STDP)

Spike-timing dependent plasticity updates weights

Idle

Main Function: Synaptic weights adapt based on spike timing. If pre-synaptic neuron fires before post-synaptic, the connection strengthens. If after, it weakens. This local learning rule runs continuously on-chip without external training.

Output Decoding & Decision

Interpret spike patterns to produce classification

Idle
